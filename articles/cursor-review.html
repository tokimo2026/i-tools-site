Cursorとは｜VSCodeにAIを載せたコードエディタ

Cursorを一言で説明すると「AIが内蔵されたコードエディタ」だ。もう少し詳しく言うと、VS Code（Visual Studio Code）というエンジニアに大人気のエディタをベースに、AIのチャット機能・コード補完・自動修正を組み込んだツールだ。

VS Codeを使ったことがある人なら、CursorはほぼそのままVS Codeの見た目・操作感で使える。VS Codeの拡張機能もそのまま使えるので、乗り換えコストが低い。

俺がCursorを知ったのは、Twitterで「CursorでWebアプリを1時間で作った」というツイートを見かけたのがきっかけだ。「1時間でWebアプリ？本当に？」と半信半疑で試してみた。

Cursorの主な機能

  • Composer（作曲家）モード：複数ファイルをまたいでAIに指示を出せる。「新しいページを追加して、ナビゲーションにリンクも追加して」みたいな複合的な指示が1回でできる
  • Chat機能：エディタの横でAIとチャットしながらコードを書ける。「この部分の意味を教えて」「このエラーの原因は何？」など、コードを見ながら相談できる
  • Tab補完：コードを書いていると、次に書くべきコードをAIが予測して提案してくれる。Tabキーを押すだけで採用できる
  • インラインEdit：コードを選択して「Ctrl+K」で、選択範囲だけをAIに修正させられる

💡 Cursorは何で動いているのか：CursorのAIモデルは複数選べる。Claude 3.5 Sonnet、GPT-4o、Geminiなど。用途によってモデルを切り替えられるのも地味に便利だ。

Claude Codeとの違い｜GUIとCLIの根本的な差

俺がCursorを試してまず思ったのは、「Claude Codeとは根本的に体験が違う」ということだ。どちらもAIを使ってコードを書くツールだが、アプローチが全然違う。

比較項目 Cursor Claude Code
インターフェース GUI（グラフィカル） CLI（コマンドライン）
操作感 VSCodeライク。視覚的 ターミナルでの対話形式
ファイル操作 エディタ内で直接確認 AIが自動でファイル編集
向いている用途 既存コードの修正・拡張 ゼロから作る・大規模変更
学習コスト 中（VSCode知識があれば低） 低〜中（ターミナル操作が必要）
AIとの会話 Chatパネルで並行表示 ターミナル内での対話
コードの可視性 常に画面で確認できる 変更後にファイルを開いて確認

簡単に言うと、Cursorは「コードを見ながらAIと作業する」ツールで、Claude Codeは「コードをAIに任せて作業してもらう」ツールだ。どちらが良いというわけではなく、作業スタイルや案件の種類によって向き不向きがある。

非エンジニア視点での正直な感想

俺の感想を正直に言うと、「最初はCursorのほうが取っつきやすかった」だ。Claude Codeはターミナルに慣れるまでが少し大変だが、Cursorはエディタを開いて使うだけなので、心理的なハードルが低い。ただ、慣れてきたらClaude Codeのほうがスピーディーに動くことも多いと感じている。

実際に作ったもの｜俺がCursorで完成させた成果物

レビューを書くにあたって、Cursorで実際に作ったものを正直に紹介する。「作れた！」という体験談だけでなく、「ここで詰まった」という話も含めて書く。

1. ToDoアプリ（JavaScript）

最初に試したのは定番のToDoアプリだ。「Cursorで作るなら何から始めるか」と調べたら大体ToDoアプリが出てくるので、素直に従った。CursorのComposerに「シンプルなToDoアプリをHTMLとJavaScriptで作って。タスクの追加・削除・完了チェックができるもの」と入力したら、3つのファイル（index.html、style.css、app.js）を一気に作ってくれた。コードはほぼそのまま動いた。所要時間は約20分。

2. クライアントのWordPressブログのカスタマイズ

副業で受けた案件で、WordPressの子テーマのCSSを修正する仕事があった。クライアントから「このボタンの色を変えたい」「スマホで見たときのメニューが崩れている」という要望を受けて、Cursorでfunctions.phpとstyle.cssを開き、チャット機能で「このCSSのここを修正して」と伝えた。エンジニアに頼んだら数時間かかりそうな修正が、30分程度で完成した。

3. シンプルな予約フォーム（HTML/CSS/JS）

個人サロンを経営している友人に頼まれて作った予約フォームだ。「名前、電話番号、希望日時を入力して送信できるフォーム。バリデーション付き」という要件をCursorに伝えたら、FormspreeというサービスのコードまでAIが提案してくれて、バックエンドなしで動くフォームが完成した。これは自分でも驚いた。

✅ Cursor初心者へのアドバイス：最初は「既存のコードを修正する」用途から始めるのがおすすめ。ゼロから作るより、動いているコードに機能を追加するほうがAIとのやり取りがスムーズで、自分も何が起きているかを理解しやすい。

メリット4つ｜Cursorが得意なこと

メリット1：コードを見ながらAIに相談できる

これがCursorの最大の強みだと思う。エディタの右側にChatパネルが開いて、左側にコードが表示された状態でAIに質問できる。「このコードの意味がわからない」「ここにバグがあると思うんだけど」と聞くと、画面に表示されているコードを参照しながら答えてくれる。Claude Codeのように「コードをコピーして貼り付けて質問する」というステップが不要なので、作業が途切れにくい。

メリット2：既存のVSCode設定がそのまま使える

俺はVSCodeをすでに使っていたので、Cursorに乗り換えたときの設定ゼロ乗り換えが助かった。テーマ、フォント、拡張機能——全部そのまま持ち込めた。VSCodeを使っている人は乗り換えコストがほぼゼロだ。

メリット3：複数ファイルをまたいだ変更が得意

Composerモードを使うと、「ヘッダーのナビゲーションに新しいページへのリンクを追加して。そのページも作って」という指示に対して、header.htmlとnewpage.htmlの両方を同時に作成・修正してくれる。これはClaude Codeでもできるが、Cursorは変更内容が画面上で差分（変更前と変更後の比較）として表示されるので、何が変わったかを視覚的に確認しやすい。

メリット4：Tabキー補完が快適

コードを書いていると、次に書きそうなコードをAIが予測してグレーで表示してくれる。それが合っていればTabキーを押すだけで採用できる。最初は「補完を蹴ることが多いな」と感じたが、1週間も使うと補完の精度に慣れてきて、Tabキーを押す頻度が増えた。タイピング量が体感で3〜4割減った気がする。

デメリット3つ｜Cursorが苦手なこと

デメリット1：無料プランの制限が厳しい

Cursorの無料プランは月に限られた回数しかAI機能（特にComposer）を使えない。副業で毎日使うと、月の半ばで制限に引っかかることがある。「あと少しで案件が終わるのに」というタイミングで制限がくるのはストレスだ。副業目的でヘビーに使うなら、有料プラン（月$20）を前提にしたほうがいい。

デメリット2：VSCodeの知識がないと最初のセットアップで詰まる

Cursorは「VSCodeライク」だが、完全に初心者向けかというとそうではない。ターミナルを開く方法、拡張機能のインストール、プロジェクトフォルダの開き方——このあたりの基礎知識がないと、最初のセットアップで詰まる可能性がある。Claude Codeのほうがある意味シンプルで、「インストールしてAPIキーを設定してコマンド打つだけ」という割り切った始め方ができる。

デメリット3：大きなプロジェクトになると文脈管理が難しい

ファイルが10個、20個と増えてくると、CursorのAIが全体像を把握しきれなくなることがある。「このファイルのあの変数が別のファイルで参照されているのに、それを無視した変更をする」みたいなことが起きる。大規模なプロジェクトでは、AIに都度「このプロジェクトの全体構成はこうだ」と教える必要があり、やや面倒だ。

⚠️ 注意：CursorのAIが出力したコードも、必ず動作確認すること。特に「こういう感じにして」という曖昧な指示で生成したコードは、意図しない動作をすることがある。小さい変更から試して、少しずつ複雑な指示に慣れていくのがベストだ。

料金プラン｜無料で十分？有料は必要？

Cursorの料金プランは以下の通りだ。

プラン 月額 主な内容 向いている人
Hobby（無料） $0 Composer 2週間トライアル後は限定回数、Tab補完一部 試しに使ってみたい人
Pro $20/月 Composer無制限、高速モデル、優先アクセス 毎日使う副業勢
Business $40/月（1人あたり） チーム機能、セキュリティ強化 チームで使う場合

正直なところ、副業で使うなら無料プランでは厳しい。2週間のトライアル期間が終わると、Composerの回数制限がかかり始める。「月に数回だけ試したい」なら無料でもいいが、毎日使うなら$20のProプランを前提にするべきだ。

ただし$20はClaude Code（API従量課金）やChatGPT Plusとほぼ同等のコストだ。「Cursor Pro + ChatGPT Plus」で月$40、「Cursor Pro + Claude Code」でも月$30〜40程度になる。副業収益でカバーできる範囲だが、最初は「Cursorだけ有料にして試す」という入り方でも良いと思う。

使い分けガイド｜Cursorを使う場面 vs Claude Codeを使う場面

1ヶ月Cursorを本格的に使った結果、俺の中で使い分けの基準が固まった。以下がそのまとめだ。

場面 おすすめ 理由
既存コードのバグ修正 Cursor コードを見ながら修正できる
新しいページ・機能をゼロから作る Claude Code ファイル作成から全部任せられる
クライアントのコードを修正する案件 Cursor 既存ファイルを開いてそのまま作業できる
丸ごとサイトを1から作る Claude Code 全体構成からAIに設計してもらいやすい
コードを学習したい Cursor コードを見ながら説明してもらえる
自動化・スクリプト作成 Claude Code ファイル操作・実行まで一気にできる
GitHubリポジトリを使った開発 Cursor Git統合がVSCodeと同等で使いやすい

✅ 俺の今の運用：Claude Codeで新しいプロジェクトの骨格を作り、細かい調整や修正はCursorでやるというハイブリッド運用に落ち着いた。作業フォルダをCursorで開いておいて、「ここを直してほしい」という局所的な修正はCursor、「全体をリファクタリングしたい」という大きな変更はClaude Codeという使い分けだ。

まとめ｜非エンジニアにCursorはおすすめか

正直な答えを言う。非エンジニアにCursorはおすすめだ。ただし「完全な初心者」にとっては最初のハードルが少しある。VSCodeを使ったことがない人は、まずClaude CodeかChatGPTから始めて、コードに慣れてからCursorに移行するのが現実的だ。

逆に「VSCodeを少し知っている」「コードは読めないけど、なんとなくわかる」というレベルの人には、Cursorは非常に使いやすいツールだ。コードを「画面で見ながら」AIと作業できる体験は、Claude Codeとは違う安心感がある。

Cursorをおすすめしたい人

  • VSCodeを使ったことがあるor興味がある人
  • 既存のWebサイトやアプリのコードを修正する副業をしたい人
  • コードを視覚的に確認しながら作業したい人
  • Claude CodeとCursorの両方を組み合わせて使いたい人

Cursorより先にClaude Codeを試すべき人

  • 完全なゼロ初心者でターミナルも触ったことがない人
  • まずはHTMLとCSSの簡単な案件から始めたい人
  • ツールを1つに絞りたい人（Claude Codeのほうがオールインワン感がある）

俺の総合評価は★★★★☆（4.0/5）だ。満点をつけなかったのは、無料プランの制限と、完全な初心者へのとっつきにくさのためだ。でもこれは慣れれば解決する問題で、使い続けるほど評価が上がるツールだと思っている。

「副業でコードを使いたいけど、どこから始めればいいかわからない」という人は、まずAIツール5選ランキングを読んでみてほしい。Cursorの立ち位置も含めて、全体像がわかる。
